!eyebrow Private Partner Concept

!privateNote This page is private partner walkthrough material for Drata and TrustSignal. It is an integration concept only, not a public product claim, not an announced native integration, and not a statement of GA partner functionality.

!summary This concept shows how Drata can remain the evidence workflow and system of record while TrustSignal performs an integrity check, returns integrity verification signals, and issues a signed verification receipt for later comparison against the previously checked artifact state.

!context Compliance artifacts can be collected correctly and still become harder to trust later if the file changes, the reference drifts, or the evidence is separated from the context that shows what was previously checked.

!integration Drata remains the workflow and system of record. TrustSignal receives the artifact or artifact reference, performs an integrity check, returns integrity verification signals, and issues a signed verification receipt that can be associated with the evidence record for later review.

!whyPartner This gives Drata teams a durable way to show what artifact state was previously checked and whether the evidence presented later still matches that state before a reviewer relies on it.

!talkTrack Start with an evidence record already living in Drata. Send the artifact hash and record reference to TrustSignal. Read back the integrity verification signals and signed verification receipt. Associate the receipt with the evidence record. Later, compare the artifact presented at review time against the previously checked state and surface any mismatch.

!productionNote In a production deployment, the integrity check request should be authenticated, the receipt stored alongside the evidence record, and later comparison invoked before a reviewer relies on a previously checked artifact.

## !!steps Drata evidence artifact

This example starts with a compliance artifact already tracked in Drata. Drata keeps the workflow state, owner, and control context.

!narration In the walkthrough, this is where the story starts: the evidence already lives in Drata before TrustSignal is introduced.

!takeaway Drata remains the system of record. TrustSignal receives stable artifact material or a stable reference plus control context.

## !!steps Integrity check request

The TrustSignal request carries the evidence reference, artifact hash, and workflow context needed to bind the integrity check to the record that Drata already manages.

!narration This is additive workflow infrastructure. The request is about the evidence record and its checked state, not about replacing Drata's workflow model.

!takeaway The request creates a stable bridge between Drata's evidence record and TrustSignal's signed verification outputs.

```json !code integrity-check-request.json
{
  "subject": {
    "system": "drata",
    "record_id": "ev_01JQ9WT9A3XK2QF1Q9M8R4A2VB",
    "control_id": "CC6.3",
    "artifact_reference": "s3://drata-evidence-bucket/access-reviews/q1-2026.csv",
    "artifact_hash": "sha256:0f3d43a0a6c70f3b4bdb5c8a8e3d8fd4c3a2d4f2836e1f8431d0a0a7984e4c8a"
  },
  "policy_profile": "drata_evidence_integrity_v1",
  "requested_by": "drata-partner-demo",
  "timestamp": "2026-03-12T15:25:10.000Z"
}
```

## !!steps Integrity verification signals

TrustSignal returns integrity verification signals in workflow-friendly language and avoids internal implementation detail.

!narration This is the operational outcome: the artifact matched the previously supplied state and a signed verification receipt was issued.

!takeaway The result is usable in the workflow immediately and points to a signed receipt that can be retained for later comparison.


## !!steps Signed verification receipt

The signed verification receipt can be associated with the Drata evidence record so a later reviewer can understand what was checked and what result was issued at the time.

!narration The receipt is the durable integrity record. It travels with the evidence record instead of replacing it.

!takeaway The receipt binds the evidence reference, the integrity verification outcome, and durable receipt metadata into a record that supports later comparison.

## !!steps Later comparison and mismatch detection

Later, a reviewer can compare the evidence presented at review time against the stored receipt context. If the artifact has changed, TrustSignal can return a mismatch instead of silently reusing the earlier result.

!narration This is where the value lands: the receipt supports later comparison against the previously checked artifact state and can surface drift before a reviewer relies on the record.

!takeaway Drift detection matters because it surfaces changes at review time without asking Drata to stop being the system of record.

```json !code later-comparison-mismatch.json
{
  "receipt_id": "rcpt_01JQ9X07S9PKS0YQG7B8W6X1CN",
  "integrity_verified": false,
  "reason": "artifact_mismatch",
  "stored_artifact_hash": "sha256:0f3d43a0a6c70f3b4bdb5c8a8e3d8fd4c3a2d4f2836e1f8431d0a0a7984e4c8a",
  "presented_artifact_hash": "sha256:7d02a6e4f5cbfd616cb50cbeeaec4d37c78f5fc0bd385b0ecff7af9afbd4ab11",
  "reviewed_at": "2026-04-07T09:12:44.000Z",
  "action": "request_fresh_evidence"
}
```
