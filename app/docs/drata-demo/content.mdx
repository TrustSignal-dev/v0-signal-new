!eyebrow Drata Integration Concept

!summary Drata can continue managing the evidence workflow and the system of record while TrustSignal adds signed verification receipts, verification signals, and later verifiable provenance checks.

!context Compliance artifacts can be collected correctly and still become harder to trust later if the file changes, the reference drifts, or the provenance details are separated from the evidence record before audit review.

!integration Drata remains the workflow and system of record. TrustSignal receives the artifact or artifact reference, returns a verification result plus a signed verification receipt, and that receipt can be associated with the evidence record for later review.

!whyDrata This gives Drata teams a durable way to show what was checked, when it was checked, and whether the artifact presented later still matches the earlier evidence state.

!talkTrack Start with an evidence item already living in Drata. Send the artifact hash and record reference to TrustSignal. Read back the verification result and signed receipt. Store that receipt with the evidence record. Later, verify the same artifact again and show whether the evidence still matches what was originally reviewed.

!productionNote In a production deployment, the verification request should be authenticated, the receipt stored alongside the evidence record, and later verification invoked before an auditor or reviewer relies on a previously checked artifact.

## !!steps Verification request

The TrustSignal request carries the evidence reference, artifact hash, and the workflow context needed to bind the verification result to the record that Drata already manages.

!narration Here I show that the integration is additive. The request is about the evidence record and its integrity context, not about replacing Drata's workflow model.

!takeaway The verification request creates a stable bridge between Drata's evidence record and TrustSignal's signed verification outputs.


## !!steps Later verification and tamper detection

Later, an auditor or reviewer can compare the evidence presented at review time with the stored receipt context. If the artifact has changed, TrustSignal can return a mismatch instead of silently reusing the earlier result.

!narration This is where the story lands. The receipt lets you prove that the artifact shown later still matches what was originally checked, or show exactly where drift appeared.

!takeaway Tamper detection is valuable because it surfaces evidence drift at review time without asking Drata to stop being the system of record.

```json !code later-verification-mismatch.json
{
  "receipt_id": "rcpt_01JQ9X07S9PKS0YQG7B8W6X1CN",
  "verified": false,
  "integrity_verified": false,
  "reason": "artifact_mismatch",
  "stored_artifact_hash": "sha256:0f3d43a0a6c70f3b4bdb5c8a8e3d8fd4c3a2d4f2836e1f8431d0a0a7984e4c8a",
  "presented_artifact_hash": "sha256:7d02a6e4f5cbfd616cb50cbeeaec4d37c78f5fc0bd385b0ecff7af9afbd4ab11",
  "reviewed_at": "2026-04-07T09:12:44.000Z",
  "action": "request_fresh_evidence"
}
```
