# Security Policy

TrustSignal takes responsible disclosure seriously. If you believe you have
found a security issue affecting the public site, documentation surface, or
associated TrustSignal properties, report it privately first.

## Reporting A Vulnerability

- Email: `security@trustsignal.dev`
- Backup contact: `info@trustsignal.dev`
- Website policy: `https://trustsignal.dev/.well-known/security.txt`

Please include:

- a concise description of the issue
- affected URL, page, or component
- reproduction steps or proof of concept
- impact assessment if known

Do not include sensitive personal data in the initial report unless strictly
necessary to explain the issue.

## Response Expectations

- We will acknowledge receipt as quickly as practical.
- We will triage the report and determine impact.
- We will work on remediation and coordinate disclosure when appropriate.

Response times may vary during pilot-stage operations, but substantive reports
will receive follow-up.

## Scope

This repository primarily covers the public TrustSignal website and related
documentation. Public site scope includes:

- `trustsignal.dev`
- public documentation pages
- pilot request and partner-access API routes in this repo

Out of scope unless explicitly invited:

- social engineering
- denial-of-service testing
- physical attacks
- third-party services not controlled by TrustSignal

## Safe Harbor

If you act in good faith, avoid privacy violations and service disruption, and
give us a reasonable opportunity to remediate before public disclosure, we will
treat your research as authorized.
